## 🔴 Critical Issues (Phải sửa ngay!)

### 1. Leakage of Sensitive Data via HTTP Logging (✅ FIXED)
- **Status:** Resolved in `NetworkModule.kt` using `BuildConfig.DEBUG` check.
- **Result:** Logs are only enabled in Debug builds; Production logs are silent.

### 2. Insecure CORS Configuration (Wide Open) (✅ FIXED)
- **Status:** Resolved in `backend/main.py`.
- **Result:** `ALLOWED_ORIGINS` is now restricted to a whitelist.

### 3. Missing Backend Caching (🆕 PERFORMANCE)
- **Status:** **Pending**. Every request triggers a new YouTube fetch.
- **Danger:** High bandwidth cost, risk of IP Ban from YouTube, High Latency.
- **Fix:** Implement `cachetools` in `backend/main.py`.

### 4. Invalid Models in Rotation (🆕 PERFORMANCE)
- **Status:** **Pending**. List includes non-existent models (Gemini 2.5/3).
- **Danger:** Guaranteed failed requests, adding 1-2s delay during rotation loop.
- **Fix:** Update `MODELS` list in `GeminiApiClient.kt`.

## 🟡 Warnings (Nên sửa)

### 1. API Key passed in Query Parameters (✅ FIXED)
- **Status:** Resolved in `GeminiApiClient.kt`.
- **Result:** API Key is now sent via `x-goog-api-key` Header.

### 2. Lack of Rate Limiting in Backend (✅ FIXED)
- **Status:** Implemented `slowapi` in `backend/main.py`.
- **Result:** Rate limited to 10 requests per minute per IP.

### 3. Redundant Prompt Building (🆕 PERFORMANCE)
- **Status:** **Pending**. Prompts are built inside the key/model nested loop.
- **Danger:** Unnecessary CPU/RAM usage during rotation.

### 4. Blocking I/O in FastAPI (🆕 PERFORMANCE)
- **Status:** **Pending**. `YouTubeTranscriptApi` call blocks the main thread.
- **Danger:** Reduced concurrency on backend.
